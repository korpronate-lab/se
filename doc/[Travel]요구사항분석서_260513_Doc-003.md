## 1. 서론

### 1.1 목적 및 범위
이 문서는 'Travel : AI 기반 여행 경비 최적화 및 실시간 항공권 가격 변동 예측' 프로젝트에서 각 요구사항이 무엇인지 조사하고, 정의하는 문서이다. 이 문서는 기능적, 비기능적, 인터페이스에 요구되는 사항들을 바탕으로 시스템의 정적, 동적 모델을 분석한다.

### 1.2 용어 정의
본 문서의 이해를 돕기 위해 사용된 모든 용어 및 약어를 설명하고 정의합니다.
| 용어 | 설명 |
| :--- | :--- |
| 동기화 (Synchronization) | 오프라인 모드에서 기록된 로컬 데이터와 서버의 실시간 데이터가 충돌 없이 일치되도록 맞추는 과정 |

| MAE | 평균절대오차(Mean Absolute Error), 예측 모델의 정확도를 평가하는 지표 |

## 2. 시스템 개요

### 2.1 소프트웨어 문맥도 (Context)

#### 2.1.1 Actor Table
| Actor | Role |
| :--- | :--- |
| 사용자 | 이 시스템을 통해 여행 예산을 계획하고 예측 결과를 확인하는 사용자를 말한다. |
| 시스템 | 사용자에게 실시간 예산 분배 결과, 항공권 가격 예측, 환율 알림을 제공하는 주체를 말한다. |
| 외부 API | 환율, 물가, 항공권 과거/현재 가격 데이터를 제공하는 외부 시스템을 말한다. |

#### 2.1.2 UseCase Diagram (텍스트 명세)
* **사용자 주도 UseCase:**
  * 여행 기본 정보(목적지, 예산, 일정)를 설정한다.
  * 환율 알림 기준을 설정한다.
  * 오프라인 상태에서 예산을 확인/기록한다.
* **시스템 주도 UseCase:**
  * 실시간 가격 및 예산을 산출한다.
  * 항공권 가격 예측을 수행한다.
  * 환전 알림을 보낸다.

### 3.3 동적 분석

**[3.3.1 여행 기본 정보를 설정한다]**
* **사용자**가 **시스템(UI)**에 목적지와 예산을 입력하고 등록을 요청한다.
* **시스템**은 입력값을 검증한 후 **여행DB**에 저장을 요청한다.
* **여행DB**는 데이터를 저장하고 성공(success) 메시지를 반환한다.
* 성공 시 **시스템**은 산출 화면으로 이동한다.

**[3.3.2 실시간 가격을 산출한다]**
* **시스템**은 **외부API**에 현지 환율 및 물가 데이터를 요청한다.
* 데이터를 수신한 **시스템**은 예산 분배 알고리즘을 실행한다.
* 산출된 결과를 **여행DB**에 업데이트한다.
* **여행DB**는 갱신 완료(success) 메시지를 반환한다.
