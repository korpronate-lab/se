## 1. 서론

### 1.1 문서 목적 및 범위
본 문서의 목적은 'Travel : AI 기반 여행 경비 최적화 및 실시간 항공권 가격 변동 예측' 프로젝트의 핵심 산출물인 '실시간 가격 계산 시스템'에 대한 테스팅을 수행하고 그 결과를 명시하는 데에 있다.
본 문서에서는 위 프로젝트의 테스트 기준, 테스트 범위, 테스트 케이스 및 테스트 결과를 정의한다. 테스트는 요구사항 정의서(Doc-002) 및 요구사항 분석서(Doc-003)에 명세된 기능적·비기능적 요구사항의 충족 여부를 사용자 관점에서 검증하는 것을 범위로 한다.

### 1.2 프로젝트 개요
본 프로젝트는 과거 항공권 가격 데이터를 분석하여 향후 가격의 상승·하락 가능성을 예측하고, 실시간 환율 및 도시별 물가 데이터를 연동하여 사용자가 설정한 예산 내에서 숙박·식비·교통비를 자동으로 분배해주는 시스템을 구축한다. 또한 항공권 구매 적기 추천, 환율 저점 환전 알림, 네트워크 단절 환경에서의 오프라인 예산 확인 및 서버-로컬 데이터 동기화 기능을 제공한다.

### 1.3 용어 정의
| 용어 | 설명 |
| :--- | :--- |
| 실시간 가격 계산 시스템 | 항공권, 환율, 현지 물가 데이터를 실시간으로 수집하여 전체 여행 예산을 산출하는 본 프로젝트의 핵심 엔진 |
| 가격 예측 모델 | 과거 항공권 시계열 데이터를 학습하여 향후 가격의 상승·하락 가능성을 확률적으로 산출하는 딥러닝 기반 모델 |
| 환율 저점 알림 | 최근 3개월 데이터 대비 현재 환율이 유리한 구간일 때 사용자에게 전송되는 푸시 알림 |
| 동기화 (Synchronization) | 오프라인 모드에서 기록된 로컬 데이터와 서버의 실시간 데이터가 충돌 없이 일치되도록 맞추는 과정 |
| 블랙박스 테스트 | 시스템 내부 구조를 고려하지 않고 입력과 출력만으로 기능의 정확성을 검증하는 테스트 기법 |
| 테스트 케이스 | 특정 기능이 요구사항대로 동작하는지 확인하기 위한 입력값·실행조건·기대결과의 집합 |
| 캐시 (Cache) | 외부 API의 응답 지연·실패에 대비하여 최근 데이터를 로컬에 임시 저장한 것 |

## 2. 테스트 개요

### 2.1 테스트 범위
테스트는 시스템 내부를 모른다는 가정 하에 블랙박스 테스트로 이루어진다. 테스트의 범위는 아래와 같다.

| 범위 | 내용 |
| :--- | :--- |
| 클라이언트 (모바일/웹 앱) | Ⅰ. 화면 동작 정확성 확인<br>Ⅱ. 요구사항 충족 확인 |
| 서버사이드 | Ⅰ. 가격 예측·예산 분배 연산 정확성 확인<br>Ⅱ. 외부 API 연동 동작 확인 |
| 데이터베이스 / 로컬 저장소 | Ⅰ. 데이터 입출력 확인<br>Ⅱ. 서버-로컬 동기화 시험 |
| 외부 API 연동 | Ⅰ. 항공권·환율·물가 데이터 수집 확인<br>Ⅱ. 장애 시 캐시 대체 동작 확인 |

### 2.2 테스트 항목 및 통과 기준

| 기능별 테스트 항목 | 통과 기준 |
| :--- | :--- |
| 회원가입 기능 | 형식이 올바르고 중복되지 않은 이메일/비밀번호만 가입에 성공한다. |
| 로그인 기능 | DB에 저장된 올바른 회원정보만 로그인에 성공한다. |
| 여행 계획 등록 기능 | 출발지·목적지·일정·예산이 유효한 경우에만 등록에 성공한다. |
| 항공권 가격 예측 기능 | 등록된 노선의 데이터를 기반으로 가격 변동 예측을 산출한다. |
| 구매 적기 추천 기능 | 예측 결과를 기반으로 추천 구매일과 예상 가격을 제시한다. |
| 실시간 환율·물가 조회 기능 | 외부 API로부터 목표 국가의 환율과 물가를 정상 조회한다. |
| 환율 저점 알림 기능 | 알림 설정과 저점 조건이 충족될 때에만 알림을 전송한다. |
| 예산 분배 기능 | 항공권 비용 차감 후 남은 예산으로 일일 평균 경비를 산출한다. |
| 오프라인 확인 기능 | 네트워크 단절 시 로컬에 저장된 예산 데이터를 열람할 수 있다. |
| 데이터 동기화 기능 | 재연결 시 로컬과 서버 데이터가 충돌 없이 동기화된다. |
| 알림 설정 기능 | 사용자가 설정한 알림 수신 여부·조건이 저장된다. |

| 비기능별 테스트 항목 | 통과 기준 |
| :--- | :--- |
| 성능 테스트 | 예측·예산 산출 결과가 2초 이내에 출력되고, 가격 예측 정확도가 90% 이상이다. |
| 보안 테스트 | 로그인하지 않은 사용자가 시스템을 이용할 수 없으며, 개인정보가 암호화되어 저장된다. |
| 국제화 테스트 | 다국어(한국어·영어) 및 현지 통화 단위 표기가 정상 지원된다. |
| API 장애 대응 테스트 | 외부 API 실패 시 캐시 데이터로 정상 대체된다. |

## 3. 테스트 케이스

### 3.1 테스트 케이스 선정
테스트 케이스는 본문 2.2의 기능별·비기능별 테스트 항목에 따라 선정하였으며, 블랙박스 테스트 기법을 기반으로 하여 시스템 내부 확인을 하지 않는 사용자 시점 테스트 케이스 선정에 중점을 둔다. 본문 3.3에서 기능별 테스트 케이스 목록을 나열한다.

### 3.2 테스트 케이스 유형
테스트 케이스 유형은 다음과 같다.
- 입력값의 유효성을 검증하기 위한 문자열 입력 테스트 케이스 (회원가입·로그인·여행 계획 등록)
- 가격 예측·추천·조회·알림·예산 분배 등 시스템 동작 확인 테스트 케이스
- 오프라인-서버 간 데이터 입력 및 동기화 확인 테스트 케이스
- 성능·보안·국제화 등 비기능 요구사항 확인 테스트 케이스

## 4. 테스트 예외사항

### 4.1 중단 기준과 재개 조건
| 중단 기준 | 재개 조건 |
| :--- | :--- |
| 시스템 input type과 다른 입력이 입력되었을 때 | 입력을 중단하고 기존의 데이터로 시스템을 재개한다. |
