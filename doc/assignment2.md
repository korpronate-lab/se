## **Software Engineering**  **2022125042 이승준**

## **(1) 사용자가 원하는 기대치는 무엇인가?**

1. 예측의 정확성 및 신뢰성  
   : 시스템이 제시하는 항공권 구매 적기 추천이 실제 비용 절감으로 이어지기를 기대합니다. 데이터 분석 결과가 실제 시장 가격 변동과 높은 일치율을 보여야 신뢰가 형성됩니다.  
2. 정보의 실시간성
   : 환율 및 물가 데이터가 실시간으로 연동되어, 예산 산출 결과가 현재 상황을 실시간으로 반영하기를 원합니다.  
4. 제약 없는 접근성
   : 해외 현지 등 네트워크가 불안정한 환경에서도 저장된 예산 데이터를 확인할 수 있는 안정적인 오프라인 기능을 기대합니다. 

## **(2) 개발자 혹은 유지보수 담당자가 원하는 기대치는 무엇인가?** 

1. 외부 API 처리의 유연함  
   : 항공권 가격, 실시간 환율, 도시별 물가 등 다양한 외부 데이터를 활용하므로, API 변경 시 대응할 수 있는 시스템이 필요합니다.  
2. 데이터 모델의 확장성  
   : 새로운 여행 국가나 노선 데이터가 추가될 때, 기존의 시스템이 효율적으로 학습하고 확장될 수 있어야 합니다.  
3. 로컬-서버 데이터 동기화의 안정성  
   : 오프라인 모드에서 기록된 데이터와 서버의 실시간 데이터가 충돌 없이 안정적으로 동기화가 이루어져야 합니다.

## **(3) 개발 조직의 관리자가 원하는 기대치는 무엇인가?** 

1. 서비스 차별화  
   : 기존 서비스가 제공하지 못하는 예산 최적화 기능을 통해 사용자를 확보하는 것을 목표로 합니다.  
2. 사용자 데이터 보안
