## 1. 서론

### 1.1 문서의 목적 및 범위
본 문서는 'Travel : AI 기반 여행 경비 최적화 및 실시간 항공권 가격 변동 예측' 프로젝트의 효율적인 추진과 체계적인 관리를 위해 작성되었다. 
* **목적:** 프로젝트의 목표, 개발 계획, 일정, 팀 구성 및 품질 관리 방안을 명확히 정의하여 이해관계자 간의 원활한 의사소통을 도모하고 프로젝트의 성공적인 완수를 지원한다.
* **범위:** 과거 항공권 가격 데이터 분석을 통한 가격 예측 모델 개발, 실시간 환율 및 물가 데이터 연동 시스템, 오프라인 예산 확인 기능 및 예산 분배 알고리즘 구현을 포함한다.

### 1.2 프로젝트 개요
본 프로젝트는 여행자가 설정한 예산 내에서 최적의 여행 계획을 세울 수 있도록 돕는 '실시간 가격 계산 시스템' 개발을 목표로 한다.
* **핵심 기능:** * Deep Learning 기반 항공권 가격 추이 분석 및 구매 적기 제시
    * 실시간 환율 모니터링을 통한 저점 구간 환전 알림 서비스
    * 네트워크 단절 상황을 대비한 로컬 데이터 저장 및 오프라인 접근 기능
    * 고정 비용(항공권) 차감 후 가용 예산 자동 분배 알고리즘

### 1.3 프로젝트 목표
이해관계자별 요구사항을 충족하여 서비스의 신뢰성과 차별성을 확보한다.
* **사용자 만족:** 실제 비용 절감으로 이어지는 높은 예측 정확도와 오프라인 환경에서의 안정적인 사용성 제공.
* **기술적 완성도:** 외부 API 변경에 유연하게 대응하는 아키텍처 구축 및 서버-로컬 간 데이터 동기화의 안정성 확보.
* **비즈니스 가치:** 기존 항공권 검색 서비스와 차별화된 종합 예산 최적화 기능을 통한 사용자 락인(Lock-in) 효과 창출 및 데이터 보안 강화.

### 1.4 용어 정의
| 용어 | 정의 |
| :--- | :--- |
| **실시간 가격 계산 시스템** | 항공권, 환율, 현지 물가 데이터를 실시간으로 수집하여 전체 여행 예산을 산출하는 핵심 엔진 |
| **항공권 가격 예측 모델** | 시계열 데이터 분석을 통해 특정 노선의 향후 가격 상승/하락 가능성을 확률로 계산하는 모델 |
| **오프라인 프로그램** | 데이터 동기화 후 네트워크 연결 없이도 기기에 저장된 예산 정보를 열람할 수 있는 기능 |
| **환율 저점 알림** | 최근 3개월 데이터 대비 현재 환율이 유리한 구간일 때 사용자에게 전송되는 푸시 알림 |

## 2. 개발 계획

### 2.1 개발 절차 모형
본 프로젝트는 **애자일(Agile) 방법론**을 채택한다. AI 모델의 성능 최적화와 사용자 피드백을 빠르게 반영하기 위해 2주 단위의 스프린트를 진행하며, 요구사항 변화에 유연하게 대응한다.

### 2.2 WBS (Work Breakdown Structure)
| 단계 | ID | 작업 명칭 | 주요 활동 |
| :--- | :--- | :--- | :--- |
| **기획/설계** | 1.1 | 시스템 설계 | DB 스키마 설계 및 API 아키텍처 정의 |
| **데이터 구축** | 2.1 | 데이터 수집 | 항공권 및 환율 Open API 연동, 데이터 크롤링 |
| **AI 모델링** | 3.1 | 가격 예측 모델 | Deep Learning 기반 시계열 예측 모델 개발 |
| **시스템 구현** | 4.1 | 백엔드 개발 | 실시간 가격 계산 및 환율 알림 로직 구현 |
| **테스트/배포** | 5.1 | 통합 테스트 | 시나리오별 예산 산출 정확도 검증 |

### 2.3 개발 내용
* **데이터 파이프라인:** 다국적 항공권 가격 및 국가별 실시간 물가/환율 API 연동.
* **예측 엔진:** 과거 데이터 학습을 통한 가격 변동 확률 산출 모델 최적화.
* **UI/UX:** 여행 예산 계획 및 오프라인 모드 최적화 인터페이스 개발.

### 2.4 개발 일정
* **1~3주차:** 요구사항 분석 및 시스템 설계
* **4~7주차:** 데이터 수집 인프라 구축 및 AI 모델 프로토타이핑
* **8~11주차:** 핵심 기능 구현 및 모델 고도화
* **12~14주차:** 통합 테스트 및 버그 수정
* **15주차:** 최종 보고 및 프로젝트 완료

## 3. 팀 구성

### 3.1 팀 구조
프로젝트 전반을 총괄하는 PM을 중심으로 AI, Backend, Frontend 담당자로 구성된 기능 중심 조직으로 운영된다.

### 3.2 역할 및 책임
* **PM:** 일정 관리, 리스크 관리, AI 모델 전략 수립.
* **AI/Backend:** 시계열 데이터 분석 및 API 서버 연동.
* **Frontend:** 모바일/웹 인터페이스 개발 및 로컬 저장소 최적화.
